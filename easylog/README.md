# LogPilot - AAOS Log Viewer v1.0.0

Android / AAOS 로그를 **실시간 수집, 파일 열기, 검색, 필터링, 내보내기** 할 수 있는 Windows 데스크톱 로그 뷰어입니다.

---

## 주요 특징

- **실시간 ADB 로그 수집** — USB로 연결된 Android 장치의 로그를 실시간으로 모니터링
- **다양한 로그 파일 열기** — `.log`, `.logcat`, `.txt`, `.zip`, `.7z` 형식 지원 (다중 파일 / 압축 내 로그 자동 추출)
- **다중 파일 타임스탬프 정렬** — 여러 파일을 동시에 열면 시간순으로 자동 정렬
- **파일 로드 진행률 + 취소** — 대용량 파일 로드 중 실시간 상태 표시 및 Cancel 기능
- **강력한 필터링** — Tag, PID, Message, Level 기반 include/exclude 필터 + 로딩 오버레이 표시
- **통합 검색** — OR(`|`) / AND(`&`) 다중 검색, 검색 하이라이트, 로딩 오버레이 표시
- **검색 히스토리** — 이전 검색어 최대 10개 자동 저장, 드롭다운 선택 및 개별 삭제 지원
- **필터 세트 관리** — 필터 규칙 저장/불러오기/색상 하이라이트
- **로그 내보내기** — 수집된 전체 로그를 50MB 분할 7z 아카이브로 저장
- **한글 로그 지원** — UTF-8, CP949, EUC-KR 인코딩 자동 감지
- **고성능 로딩** — Regex 제거, 병렬 파일 로드, 메모리 최적화
- **스풀 자동 정리** — 24시간 이상 된 임시 스풀 파일 앱 시작 시 자동 삭제

---

## 실행 방법

1. 배포 폴더에서 `LogPilot.exe`를 더블클릭하여 실행합니다.
2. ADB 실시간 로그를 사용하려면 PC에 `adb`가 설치되어 있어야 합니다.
3. 로그 내보내기(Export) 기능을 사용하려면 `7z`(7-Zip)가 설치되어 있어야 합니다.

## 사용법

### 1. 실시간 로그 수집 (ADB Live)

1. Android 장치를 USB로 PC에 연결합니다.
2. **Discover Devices** 를 클릭하여 장치를 탐색합니다.
   - 장치가 1개뿐이면 자동 선택됩니다.
3. **▶ Start Live** 를 클릭하면 실시간 로그 수집이 시작됩니다.
   - 시작 시점 기준 최근 30초 로그가 먼저 표시된 후 새 로그가 이어집니다.
4. **■ Stop Live** 를 클릭하면 수집이 중단됩니다.

**스크롤 제어**
- 마우스 휠 또는 로그 행 클릭 → 자동 스크롤 해제
- `End` 키 → 자동 스크롤 재활성화
- `Home` 키 → 로그 최상단으로 이동
- `PageUp` / `PageDown` / `↑` / `↓` → 페이지/행 단위 이동

### 2. 로그 파일 열기 (Open Log)

- **Open Log** 버튼 → 파일 선택 다이얼로그에서 `.log`, `.logcat`, `.txt`, `.zip`, `.7z` 파일을 선택합니다.
- **다중 파일 선택** 이 가능합니다. 분할된 여러 로그 파일을 한번에 로드할 수 있습니다.
- **Drag & Drop** — 파일을 앱 창으로 끌어다 놓아도 열 수 있습니다.
- 대용량 파일 로드 시 로딩 상태가 우측 상단에 표시됩니다.

### 3. 검색

- 상단 검색창에 검색어를 입력하고 **Enter** 또는 **Find** 를 클릭합니다.
- `Ctrl+F`로 검색창에 바로 포커스할 수 있습니다.
- **검색 대상**: Tag, Message, PID를 통합 검색합니다.
- **검색 중 로딩 오버레이**가 표시되어 진행 상태를 확인할 수 있습니다.

**다중 검색 지원**
| 구분자 | 의미 | 예시 |
|--------|------|------|
| `\|` | OR (하나라도 포함) | `error\|crash` |
| `&` | AND (모두 포함) | `audio&timeout` |

- `&`가 `|`보다 우선 적용됩니다. 예: `error|audio&timeout` = `error` OR (`audio` AND `timeout`)
- 검색 결과는 하단 **Search Results** 패널에 표시되며, 결과 행을 클릭하면 메인 로그에서 해당 위치로 이동합니다.
- **Search All Logs** 체크 시 필터와 무관하게 전체 로그에서 검색합니다.

**검색 히스토리**
- 검색을 수행하면 검색어가 자동으로 히스토리에 저장됩니다 (최대 10개).
- 검색창에 포커스하면 이전 검색어 드롭다운이 표시됩니다.
- `↑` / `↓` 화살표 키로 히스토리를 탐색하고 선택할 수 있습니다.
- 각 히스토리 항목의 `✕` 버튼으로 개별 삭제가 가능합니다.
- 히스토리는 앱 재시작 후에도 유지됩니다.

### 4. 필터링

좌측 **Filter Rules** 패널에서 필터 규칙을 관리합니다.

**필터 규칙 추가/편집**
1. **Add** 버튼 클릭 → 필터 편집 팝업이 열립니다.
2. Include 조건: Text, Tag, PID
3. Exclude 조건: Text, Tag, PID
4. Level 필터: V / D / I / W / E / F 개별 선택
5. 색상 지정: 필터에 매칭되는 로그 행에 배경색 표시
6. 다중 검색어 구분: `|` 사용 (예: `CarService|AudioService`)

**필터 적용 시 로딩 표시**
- 필터 ON/OFF 전환, 필터 추가/삭제/편집 시 "Applying filters..." 로딩 오버레이가 표시됩니다.
- 대용량 로그에서 필터 적용 중에도 앱이 멈추지 않았음을 사용자에게 알려줍니다.

**필터 조작**
- **Enable / Disable** — 필터 규칙 활성화/비활성화
- **체크박스** — 여러 규칙을 일괄 선택하여 Enable/Disable
- **Select All** — 모든 규칙 전체 선택/해제
- **↑ / ↓ 버튼** 또는 **Drag & Drop** — 필터 우선순위 변경
- **더블클릭** — 해당 규칙 편집 팝업 열기
- **우클릭** → Add / Edit / Delete 메뉴

### 5. 필터 세트 관리

필터 규칙 묶음을 파일로 저장하고 불러올 수 있습니다.

| 버튼 | 기능 |
|------|------|
| **New Set** | 새 빈 필터 세트 생성 (미저장 시 저장 확인 팝업) |
| **Load Set** | 저장된 `.json` 필터 세트 파일 불러오기 |
| **Save Set** | 현재 필터 세트 저장 |
| **Save As** | 다른 이름으로 저장 |

- 필터 세트는 `LogFilter/` 폴더에 JSON 형식으로 저장됩니다.

### 6. 로그 복사

| 방법 | 동작 |
|------|------|
| 로그 행 선택 → `Ctrl+C` | 선택된 행 전체 복사 |
| 우클릭 → **Copy Rows** | 선택된 행 전체 복사 |
| 우클릭 → **Copy Message Only** | Message 열만 복사 |
| Preview 패널에서 드래그 → `Ctrl+C` | 선택한 텍스트 일부만 복사 |

### 7. Preview 패널

좌측 하단 **Preview** 영역에서 선택한 로그 행의 상세 정보를 확인할 수 있습니다.

- 로그 행 또는 검색 결과 행을 클릭하면 해당 Message가 Preview에 표시됩니다.
- 헤더에 #, Time, Level, Tag, PID, TID 요약이 표시됩니다.
- Preview 내에서 텍스트를 드래그하여 일부만 복사할 수 있습니다.

### 8. 로그 내보내기 (Export)

1. **Export Logs** 버튼을 클릭합니다.
2. 저장 위치와 파일명을 선택합니다.
3. 수집된 전체 로그가 **50MB 단위로 분할**되어 `.7z` 아카이브로 저장됩니다.
4. 분할된 각 파일명에는 마지막 로그의 시각이 포함됩니다.

### 9. 로그 지우기 (Clear)

- **Clear Logs** 버튼을 클릭하면 현재 화면에 출력된 로그가 즉시 비워집니다.
- 실시간 수집 중이라면 이후 새로 수집되는 로그부터 다시 표시됩니다.

### 10. UI 설정

| 설정 | 방법 |
|------|------|
| **폰트 종류** | 상단 메뉴에서 폰트 선택 (Segoe UI, Consolas, Malgun Gothic 등) |
| **폰트 크기** | A- / A+ 버튼으로 로그 출력 영역 크기 조절 |
| **컬럼 너비/순서** | 드래그로 조정 — 앱 재시작 시에도 유지됩니다 |
| **패널 크기** | 좌측 필터 패널, 하단 검색 결과 패널의 경계를 드래그하여 조절 |

## 사전 요구 사항

| 항목 | 용도 | 필수 여부 |
|------|------|-----------|
| Windows 10 이상 (x64) | 앱 실행 | 필수 |

## 지원 로그 형식

- Android `logcat -v threadtime` 표준 형식
- 연도 접두사 형식 (`YYYY-MM-DD HH:mm:ss.fff`)
- 마이크로초 타임스탬프
- UID 필드 포함 형식
- 태그 없는 커널 로그

- **앱 이름 변경**: ALV → **LogPilot** 으로 리브랜딩

### 성능 최적화
- **메모리 최적화**: Tag 인터닝, LogRowModel 경량화, Snapshot 복사 제거, LINQ 핫패스 제거
