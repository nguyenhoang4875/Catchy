## 변경 내역

### 1. 로그 로딩 / 실시간 성능 개선
- **파일 점진적 렌더링**: 대용량 파일 로드 시 배치 단위로 화면에 즉시 반영하여 체감 속도 향상
- **라이브 배치 append**: 실시간 로그 수신을 배치 이벤트로 처리하여 고빈도 로그 환경에서 UI 버벅임 완화
- **단일 패스 필터 + 검색 취소**: 필터/검색 파이프라인을 단일 패스로 최적화하고, 검색 재진입 시 이전 작업을 취소
- **다중 파일 K-way 병합**: 여러 파일 로드 시 전체 정렬 대신 파일별 정렬 + K-way 병합으로 메모리/시간 효율 개선

### 2. 다중 파일 로드 시 동일 ms 타임스탬프 순서 보존 (버그 수정)
- 여러 로그 파일을 동시에 로드할 때, **밀리초 단위까지 동일한 타임스탬프**를 가진 로그 라인들의 순서가 raw data와 다르게 뒤죽박죽되던 문제 수정
- 원인: 파일별 정렬에 불안정 정렬(`List<T>.Sort`)을 사용하여 동률 레코드의 라인 순서가 보장되지 않음
- 해결: 파일별 정렬을 `(Timestamp, RowId)` 2차 키 기반 안정 정렬로 변경하여 파일 내 원래 라인 순서를 정확히 보존
- 회귀 테스트(`TestMultiFileSameTimestampOrderingAsync`) 추가

### 3. About 팝업 사용성 개선
- About 다이얼로그의 텍스트를 **선택/복사 가능**하도록 변경
- 문의 이메일 **복사 버튼** 추가

### 4. 툴바 / UX 개선
- 상단 툴바 구성 및 로그 로드 관련 UX 보완
